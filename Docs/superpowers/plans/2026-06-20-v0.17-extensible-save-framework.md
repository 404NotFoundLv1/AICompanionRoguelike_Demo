# v0.17 Extensible Save Framework Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist the AI companion relationship across application restarts through a versioned, extensible default-slot save framework with visible home-scene feedback.

**Architecture:** Serializable save DTOs form a versioned root document with an independent relationship section. A JSON file store owns disk IO, a service maps save DTOs to runtime state, and a persistent coordinator loads before the first scene and autosaves when the relationship snapshot changes. Gameplay components never read or write files directly.

**Tech Stack:** Unity 6, C#, `JsonUtility`, `System.IO`, NUnit EditMode tests, Unity MCP scene configuration.

- [ ] **Step 1: Add the root and relationship DTOs**

The root owns schema version and timestamp; the relationship section owns only durable relationship data:

```csharp
[Serializable]
public sealed class GameSaveData
{
    public const int CurrentVersion = 1;
    public int saveVersion = CurrentVersion;
    public string savedAtUtc = string.Empty;
    public RelationshipSaveData relationship = new RelationshipSaveData();
}

[Serializable]
public sealed class RelationshipSaveData
{
    public bool hasData;
    public int trust;
    public int affection;
    public List<RelationshipMemoryTagScore> memoryTags = new List<RelationshipMemoryTagScore>();
}
```

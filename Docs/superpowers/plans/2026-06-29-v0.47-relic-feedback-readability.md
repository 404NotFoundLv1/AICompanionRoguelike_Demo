# v0.47 Relic Feedback Readability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the existing lightweight relic system readable in play: relic choices stand out, relic pickup/effects are shown as explicit feedback, and Sync Mark has an in-world marker.

**Architecture:** Keep relic gameplay in `RunManager` and `RunRelicRules`, add a small feedback surface that UI can read without knowing relic internals, and make `RelicSyncMarkTarget` own the simple visual marker on enemies. Reuse IMGUI panels already used by rewards and room feedback; do not add a new UI framework.

**Tech Stack:** Unity 6000 C#, EditMode tests with NUnit, existing Unity MCP scene configuration.

---

### Task 2: Add Relic Feedback Lines

**Files:**
- Modify: `Assets/_Game/Scripts/Roguelike/RunRelicRules.cs`
- Modify: `Assets/_Game/Scripts/Roguelike/RunManager.cs`

- [ ] **Step 1: Implement readable labels**

Add rule helpers:
- `GetChoicePrefix(RunRelicType relicType)` returns `RELIC`;
- `GetPickupBanner(RunRelicType relicType)` returns `Relic acquired: <title>`;
- `GetEffectBanner(RunRelicType relicType, string detail)` returns `<title>: <detail>`.

- [ ] **Step 2: Surface current feedback**

Add `RunManager.LastRelicBannerMessage` and set it when:
- `AcquireRelic` succeeds or repeats;
- First Aid triggers on combat-room entry;
- Field Backpack adds the extra Prepare supply.

Expected: UI and tests can read one concise current relic line without parsing the room feedback paragraph.

### Task 4: Add Sync Mark In-World Marker

**Files:**
- Modify: `Assets/_Game/Scripts/Roguelike/RelicSyncMarkTarget.cs`

- [ ] **Step 1: Create a simple child marker**

When `MarkByCompanion` is called, create or enable a child GameObject named `SyncMarkVisual`, place it above the target, and give it a small cyan SpriteRenderer using Unity's built-in square sprite.

- [ ] **Step 2: Hide marker when cleared**

`ClearMark` disables the marker. `OnDisable` should also disable it so pooled/disabled targets do not show stale visuals.

Expected: in play, an AI-marked enemy has a clear cyan marker without requiring prefab edits for every enemy.

# v0.49 Room Interaction Visuals Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Unify the visual language for route portals, shops, and rest points so current objectives map to obvious in-world interactables.

**Architecture:** Add one lightweight `RoomInteractionVisualCue2D` component that owns shared cue state: role label, visible/available/highlighted state, color, and a child marker. Existing `NextRoomChoicePortal`, `SupplyShopInteractable`, and `SafeRestInteractable` keep their behavior, but delegate visual state to the cue component and use `RunManager.CurrentNextStepLabel` to decide when to highlight.

**Tech Stack:** Unity 6000 C#, simple `SpriteRenderer` visuals, existing IMGUI prompts, NUnit EditMode tests, Unity MCP verification.

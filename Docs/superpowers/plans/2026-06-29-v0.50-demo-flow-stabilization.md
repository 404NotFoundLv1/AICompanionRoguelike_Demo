# v0.50 Demo Flow Stabilization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the current demo run readable and hard to soft-lock from home expedition through route choices, support rooms, boss completion, and return-home readiness.

**Architecture:** Keep the existing `RunManager` as the single source of truth for run flow state. Add a small demo-flow status surface that reports whether the current state has an actionable next step, then display that line in `StatusFeedbackUI` and verify the scene is configured for a short complete run.

**Tech Stack:** Unity 6000.4, C#, NUnit EditMode tests, Unity MCP scene configuration.

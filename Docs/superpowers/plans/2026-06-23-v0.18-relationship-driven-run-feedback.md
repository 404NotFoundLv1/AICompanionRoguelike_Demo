# v0.18 Relationship-Driven Run Feedback Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make persisted Trust, Affection, and memory tags visibly and mechanically affect the next combat run through relationship tiers, natural opening feedback, and QTE request cooldowns.

**Architecture:** A pure Memory-domain evaluator derives a bond tier, dialogue tone, dominant memory, and QTE cooldown multiplier from the existing relationship snapshot. A Companion component turns that profile into a local opening line and broadcasts it, while the existing status UI displays it. `CompanionQTERequester` consumes only the evaluator's cooldown multiplier, keeping combat deterministic and independent from dialogue text.

**Tech Stack:** Unity 6, C#, NUnit EditMode tests, IMGUI status feedback, Unity MCP prefab and scene validation.

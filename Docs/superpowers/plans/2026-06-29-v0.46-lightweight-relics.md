# v0.46 Lightweight Relics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a small, playable relic system that links combat rewards, shop purchases, safe-room preparation, combat effects, HUD readability, and run settlement.

**Architecture:** `RunManager` owns the current run relic inventory and surfaces relic reward choices. Focused relic rule files define definitions, labels, and combat modifiers so player/companion combat can integrate without bloating `RunManager`. Existing `StatusFeedbackUI` and `HomeRunSummaryUI` display current and completed run relics.

**Tech Stack:** Unity 6000.4, C#, NUnit EditMode tests, existing IMGUI panels, existing RunManager reward/shop/safe-room flow.

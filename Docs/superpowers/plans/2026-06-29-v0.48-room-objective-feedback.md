# v0.48 Room Objective Feedback Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every room state readable during a run: current objective, progress, and next step should be visible without relying on long debug-style room text.

**Architecture:** `RoomManager` will expose the initial enemy count for the current room. `RunManager` will own readable objective/progress/next-step labels because it already knows reward, shop, safe-rest, route-choice, and completion state. `StatusFeedbackUI` will display those labels in the existing run status panel.

**Tech Stack:** Unity 6000 C#, existing IMGUI HUD, NUnit EditMode tests, Unity MCP verification.

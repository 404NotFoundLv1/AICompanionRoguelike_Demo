# v0.35 Run Growth Route Design

## Goal

Close the in-run growth loop. Reward choices should not only improve stats; repeated choices should form a readable run route that nudges future rewards and gives the AI companion something to react to.

## Scope

- Determine the current run growth route from applied reward categories.
- A route becomes active when one category reaches two selected rewards.
- Route level equals the number of selected rewards in that category.
- Reward generation lightly favors the active route while preserving other options.
- Show the active route in the status panel.
- Trigger a short AI speech bubble when the route changes.

## Out of Scope

- No permanent route unlocks.
- No save schema change.
- No new reward rarity tier.
- No hard lock into one route.
- No route-specific new active skills yet.

## Architecture

`RunManager` continues to own the run reward state. v0.35 extends the existing v0.34 category counters into a route evaluation rule: the category with the highest count becomes active at count two. Ties keep the previous active route when possible, otherwise the route is still considered forming.

Reward preparation first attempts to include one choice from the active route category. It then uses the v0.34 category mix to keep other routes visible. This gives the player a route continuation option without removing pivots.

`StatusFeedbackUI` reads a new `CurrentGrowthRouteLabel` from `RunManager`. AI route feedback reuses `CompanionSpeechBubbleUI` so no new UI object is required.

## User Feedback

- Before route activation: `Growth Route: forming`.
- After route activation: `Growth Route: Counterplay Lv2`, `Growth Route: Player Lv3`, etc.
- AI companion says a short route-specific line when a route first activates or changes.

## Testing

EditMode tests cover:

- Selecting two rewards in one category activates a route label.
- Preparing rewards after route activation includes a reward from the active route category.
- Switching to another stronger category updates the route and route label.
- Existing reward readability tests remain green.

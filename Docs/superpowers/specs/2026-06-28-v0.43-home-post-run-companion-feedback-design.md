# v0.43 Home Post-Run Companion Feedback Design

## Goal

Make returning home feel like the AI companion remembers the last run and responds to it.

## Scope

- Store the AI tactic used in the last run inside `RunSessionSummary`.
- The home companion dialogue shows a clear last-run recap: result, room progress, last room, AI tactic, and any companion/Boss feedback already recorded by combat systems.
- The existing three home replies remain the interaction surface: thank support, discuss tactics, and keep distance.
- Choosing a reply still writes Trust, Affection, and a memory tag, but the dialogue copy should read like a response to the last run.

## Approach

Extend the existing `HomeCompanionDialogue` rather than adding a second home dialogue component. Capture `CompanionRunBuildState.CurrentTendency` when the run summary is created, then use deterministic formatting helpers for the recap line and tactic label so EditMode tests can verify the behavior without depending on GUI rendering.

## Data Flow

Home expedition selection writes `CompanionRunBuildState.CurrentTendency`. `RunSessionState.CreateSummary()` copies that value into `RunSessionSummary`. In HomeScene, `HomeCompanionDialogue.BuildDialogueText()` reads `RunSessionState.LastSummary`, seeds relationship values from the summary, displays the recap, and applies the chosen reply through the existing `CompanionRelationship.ApplyMemoryEvent()` path.

## Testing

EditMode tests cover:

- A run ended after selecting Link stores Link in `RunSessionSummary`.
- Home dialogue recap includes run result, room progress, last room, and AI tactic.
- Discuss Tactics writes a Reliable memory and reads like a post-run review.

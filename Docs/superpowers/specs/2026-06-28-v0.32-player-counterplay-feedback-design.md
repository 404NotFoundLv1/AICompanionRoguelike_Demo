# v0.32 Player Counterplay Feedback Design

## Goal

Make the current enemy attack patterns feel fair by giving the player readable counterplay and clear feedback when the counterplay works.

## Scope

- Reuse the existing player dash input and movement fields.
- Add a player damage modifier that cancels enemy damage while dash invincibility is active.
- Add short post-hit invulnerability so repeated enemy hits cannot instantly stack.
- Emit lightweight feedback for dash starts, successful dodges, invulnerability blocks, and Guard opening hits.
- Let the AI companion surface tactical hints through the existing speech bubble path.
- Configure the playable battle scene so the player prefab has the required counterplay component.

## Out of Scope

- No stamina system.
- No animation controller overhaul.
- No new player skill tree.
- No slow-motion or cinematic effect.
- No changes to QTE input rules.

## Architecture

`PlayerCounterplayFeedback` lives on the player and implements `IDamageModifier`. It reads `PlayerMovement2D.IsInvincible` and a short internal post-hit timer to decide whether incoming enemy damage should be canceled. It exposes events and last-message state for UI/tests.

`EnemyProjectile2D` continues to move and resolve normally, but if it intersects the player while damage is canceled, it reports a miss-style resolution instead of pretending the player was hit. Direct enemy attacks also flow through `HealthComponent.TakeDamage`, so the same modifier protects against melee and projectile damage.

`PlayerCombat2D` emits a hit event after damage is applied. `PlayerCounterplayFeedback` listens for player hits against `EnemyAttackPattern2D` and reports Guard opening hits when the target is vulnerable.

## User Feedback

- Dash start: local feedback message can read "Dash: invulnerable".
- Successful projectile/damage dodge: "Perfect dodge: no damage".
- Post-hit block: "Recovering: avoided follow-up damage".
- Guard opening hit: "Counter hit: Guard opening punished".
- Companion lines use the existing bubble system and should say concise tactical hints such as "AI: Dash through the projectile." and "AI: Guard opening. Hit now."

## Testing

EditMode tests cover:

- Dash invincibility cancels enemy damage.
- Post-hit invulnerability blocks a rapid second hit.
- A projectile resolving into a dashing player does not reduce HP.
- Hitting a vulnerable Guard produces counter feedback.
- Battle scene/player configuration includes the counterplay component.

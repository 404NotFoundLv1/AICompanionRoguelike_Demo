# v0.30 AI Companion Target Priority Design

## Goal

Make the companion recognize Melee, Ranged, and Guard enemies, select targets for tactical reasons instead of distance alone, and explain important target changes through the existing speech bubble.

## Design

- `CompanionTargetPriorityRules` is a pure scoring layer. It considers enemy archetype, companion distance, player distance, active enemy warnings, current Build tendency, and a small current-target bonus.
- `CompanionSensor` remains the single target authority. It evaluates every valid detected enemy with the rules and publishes the selected target plus its decision reason.
- `CompanionCombatDialogueController` subscribes to target changes and reuses `CompanionSpeechBubbleUI`; no second combat UI is introduced.
- Balanced companions favor ranged pressure. Guardian companions can override that choice to intercept an immediate player threat. Suppressor companions strongly favor ranged threats. Link companions gain extra interest in Guard enemies as coordination targets.
- Bosses and enemies without `EnemyArchetype2D` use the Melee fallback, preserving existing behavior.

## Feedback

Target-change lines are short bilingual tactical callouts. They are sent at normal tactical priority and remain subject to the existing global dialogue cooldown so frequent scans cannot spam the screen.

## Verification

- EditMode rule tests cover ranged preference, Guardian interception, Build-specific weighting, and decision text.
- Sensor integration tests verify that a farther Ranged enemy can beat a nearer Melee enemy and expose the chosen reason.
- Full EditMode regression plus a Unity Play Mode smoke check confirms target selection and speech-bubble feedback in a spawned combat room.

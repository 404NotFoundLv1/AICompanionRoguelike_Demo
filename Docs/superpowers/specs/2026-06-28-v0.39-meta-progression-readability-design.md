# v0.39 Meta Progression Readability Design

## Goal

Make the existing home meta progression loop readable in play: players can see their Core Fragments and permanent upgrade levels in the home, understand upgrade purchase results immediately, and see which permanent upgrades were applied when a new run starts.

## Scope

- Add a lightweight home HUD for Core Fragments and the three permanent upgrade levels.
- Improve the home upgrade station with visible purchase success/failure feedback.
- Add a short battle-scene message when saved permanent upgrades are applied.
- Keep the upgrade model unchanged: Player Max HP, Player Damage, AI Support Cooldown.

## Approach

Use immediate-mode GUI to match the current prototype UI. Add small formatting helpers that return deterministic strings, so tests can lock down the player-facing information without depending on screen rendering. Home HUD and upgrade station should clamp panels to the Game view, following the v0.38 station behavior.

## Data Flow

`MetaProgressionState` remains the source of truth. The home HUD reads it every frame. The upgrade station purchases through `MetaProgressionState.TryPurchaseUpgrade`, stores a short visible result message, and autosave continues through the existing `StateChanged` event. `RunManager.StartRun` applies saved upgrades and records a temporary feedback line in the room feedback panel.

## Testing

EditMode tests should cover:

- Home HUD status text includes Core Fragments and all three upgrade levels.
- Upgrade station purchase feedback distinguishes success from insufficient fragments.
- Run start feedback text lists the applied permanent upgrade levels.

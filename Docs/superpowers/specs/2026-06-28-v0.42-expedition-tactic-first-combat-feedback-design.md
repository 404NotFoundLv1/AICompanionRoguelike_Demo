# v0.42 Expedition Tactic First Combat Feedback Design

## Goal

Make the AI tactic chosen in home expedition preparation immediately readable during the first combat room.

## Scope

- The selected Guardian, Suppressor, or Link tactic produces a first-combat AI speech bubble line when the first non-branch room starts.
- The room feedback panel includes the selected tactic and a short first-fight effect hint when combat begins.
- Room clear feedback reminds the player that the chosen tactic can shape later Build rewards.
- The run status panel exposes a short AI tactic plan line in addition to the existing AI Build summary.

## Approach

Keep the behavior on the existing run-feedback path. `CompanionSkillTendencyRules` will own the readable tactic copy so the speech bubble, room feedback, and HUD agree. `CompanionCombatDialogueController` will speak the first-room launch line once. `RunManager` will append tactic feedback to room start and room clear text. `StatusFeedbackUI` will draw a tactical plan line from the same rules.

## Data Flow

Home expedition selection writes `CompanionRunBuildState.CurrentTendency`. Combat systems read that state. `RoomManager.RoomStarted` triggers `CompanionCombatDialogueController`; `RunManager.AdvanceToRoom` and `RunManager.HandleRoomCleared` build visible panel messages; `StatusFeedbackUI` reads the current tactic every frame for its status panel.

## Testing

EditMode tests cover:

- First combat room speech bubble names the selected Link tactic and its QTE effect.
- Room start and room clear feedback include the selected tactic and reward influence.
- Status panel tactic plan line names the selected Suppressor tactic.
- Tactic launch copy exists for Guardian, Suppressor, and Link.

# v0.41 Expedition AI Tactics Design

## Goal

Let the player choose the AI companion's run tactic during the home expedition preparation step, then carry that choice into the next combat run.

## Scope

- The home exit preparation panel shows three tactical choices: Guardian, Suppressor, and Link.
- Pressing `1`, `2`, or `3` while the panel is open selects the current run AI tactic.
- Opening the preparation panel starts a fresh run-build selection and defaults to Guardian so the player can confirm immediately.
- Confirming the expedition keeps the selected tactic for the loaded combat scene.
- Directly playing the combat scene still shows the existing combat Build choice UI when no tactic was selected from home.

## Approach

Extend `HomeExitPortal`, because it already owns the home-to-combat transition and v0.40 preparation UI. Reuse `CompanionRunBuildState` and `CompanionSkillTendencyRules` so combat support, QTE frequency, target priority, and HUD text continue to read the same state.

## Data Flow

`HomeExitPortal.OpenPreparationPanel()` resets stale run-build state, selects Guardian as the default, and opens the panel. `HomeExitPortal.SelectPreparationTendency()` normalizes and writes the selected tendency to `CompanionRunBuildState`. The combat scene preserves that state by disabling the old `CompanionBuildChoiceUI` reset-on-enable behavior in scene configuration.

## Testing

EditMode tests should cover:

- Opening expedition preparation resets stale run-build state and defaults to Guardian.
- Selecting Link or Suppressor from preparation updates `CompanionRunBuildState`.
- Preparation lines expose the `1/2/3` tactical choices and show the current selection.

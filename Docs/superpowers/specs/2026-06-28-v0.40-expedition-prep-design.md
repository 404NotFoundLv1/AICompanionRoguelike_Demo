# v0.40 Expedition Prep Design

## Goal

Turn the home exit portal into a deliberate expedition preparation step. The player should see what permanent upgrades and AI relationship state will be carried into the next run before leaving the home.

## Scope

- Pressing the home portal interaction key opens a confirmation panel instead of immediately loading battle.
- The confirmation panel shows permanent upgrades, AI bond tier, relationship values, dominant memory, and current AI Build tendency.
- Pressing confirm starts the run and loads the battle scene.
- Pressing cancel closes the panel and keeps the player in the home.

## Approach

Extend `HomeExitPortal` because it already owns the transition to battle and run-session start. Keep the UI immediate-mode for consistency with the prototype. Add deterministic formatting helpers so tests can validate the information shown in the panel without relying on rendering.

## Data Flow

`HomeExitPortal` reads `MetaProgressionState`, `CompanionRelationshipState`, `CompanionRelationshipProfile`, and `CompanionRunBuildState`. It does not change those systems. Confirming the panel calls the existing `EnterBattle()` method, preserving the current scene-transition flow.

## Testing

EditMode tests should cover:

- Preparation lines include permanent upgrade levels.
- Relationship lines include bond tier, trust, affection, dominant memory, and build tendency.
- The portal opens confirmation before loading and can cancel confirmation without transitioning.

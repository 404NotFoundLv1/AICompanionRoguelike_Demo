# v0.45 Safe Room Rest Point Design

## Goal

Turn SafeRoom from an automatic heal room into a readable rest room with a visible interactable rest point and one meaningful choice before the player continues the route.

## Player Flow

1. Player enters a SafeRoom.
2. No healing happens automatically.
3. The next-route portal stays available so the player can skip resting.
4. A visible rest point appears in the room.
5. Interacting with the rest point opens three options:
   - Rest: recover HP.
   - Talk: gain a small AI relationship and memory benefit.
   - Prepare: gain 1 supply for the next route.
6. Choosing one option closes the rest UI and returns to the route-choice flow.
7. The rest point can only be used once per SafeRoom.

## Rules

- SafeRoom has no enemies and remains safe.
- Recovery modifier still affects the Rest option heal amount.
- Talk writes `RelationshipMemoryTag.Reliable`.
- Prepare grants 1 supply as the smallest useful preparation effect for this prototype.
- Esc closes the rest UI without spending the one rest choice.

## Unity Setup

- `SampleScene` gets one `SafeRestPoint` world object.
- `SafeRestPoint` is visible only in SafeRoom.
- `SafeRestChoiceUI` is present in the scene and only renders while a rest draft is open.

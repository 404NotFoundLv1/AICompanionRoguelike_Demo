# v0.34 Reward Readability Design

## Goal

Make room rewards easier to understand during a run. The player should see what kind of growth each reward represents, what value will change, and how the current run is leaning.

## Scope

- Add reward categories: Player, AI, Counterplay, Survival, and Build.
- Add a compact value preview to reward choices, such as `Damage: 35 -> 42`.
- Keep the current IMGUI reward panel, but show category and preview lines.
- Make reward choice generation less scattered by guaranteeing a small category mix when enough choices are available.
- Show a run growth summary in the status panel.

## Out of Scope

- No new art pass for reward cards.
- No permanent skill tree.
- No rarity system.
- No weighted long-term balance model.
- No save schema change for reward counts.

## Architecture

`RunRewardChoice` grows from title/description only into a readable reward payload. It keeps backward-compatible constructors and adds category, category label, preview line, and growth tag.

`RunManager` remains the source of reward creation and application. It owns category rules, builds preview lines from currently available runtime components, prepares a more stable category mix, and tracks applied reward category counts for the active run.

`RoomRewardChoiceUI` stays lightweight and only renders data already prepared by `RunManager`. `StatusFeedbackUI` reads `RunManager.CurrentGrowthSummaryLabel` and displays it with the existing run status lines.

## User Feedback

- Reward buttons show `[Category] Title`.
- Reward descriptions remain short.
- Each reward choice shows one value preview line when runtime data exists.
- The status panel shows `Growth: Player X | AI Y | Counterplay Z | Survival W | Build V`.

## Testing

EditMode tests cover:

- Reward choices expose category labels and current-value preview lines.
- Prepared Battle Room rewards contain at least one Counterplay option and more than one category.
- Applying rewards updates the run growth summary.
- Existing reward and route tests remain green.

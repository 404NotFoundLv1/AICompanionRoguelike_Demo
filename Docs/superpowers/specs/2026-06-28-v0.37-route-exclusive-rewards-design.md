# v0.37 Route Exclusive Rewards and Run Summary Design

## Goal

Make an active growth route feel like a real build identity. Once a route is formed, the next reward drafts can surface a route-exclusive reward, and the end-of-run summary records the main route, route effect, and exclusive reward count.

## Scope

This version stays inside the current reward and run-completion loop. It does not add new room events, new scenes, permanent unlocks, or save-slot data. The feature is a small playable layer on top of v0.35 and v0.36.

## Reward Design

Add one runtime reward type, `GrowthRouteSpecialization`. It is dynamic: its title, category, preview, and effect are based on the currently active growth route.

- Player route: route-exclusive reward deepens player attack damage.
- AI route: route-exclusive reward further lowers companion attack cooldown.
- Counterplay route: route-exclusive reward improves dodge counter duration and counter damage.
- Survival route: route-exclusive reward improves bond rescue retained HP.
- Build route: route-exclusive reward adds temporary build route bonus level.

The reward appears only after a growth route is active. It is offered before the normal same-category route reward so the player can clearly continue the route.

## Runtime State

`RunManager` owns route specialization counts per category. The existing v0.36 route effect properties include these counts, so existing combat components keep querying the same APIs.

When `GrowthRouteSpecialization` is selected, the active route's specialization count increases and the reward also counts as growth for that route. This keeps the route label, effect value, and reward drafting aligned.

## End Summary

`RunSessionState` records a growth-route summary before victory completion. `RunSessionSummary` exposes:

- whether a growth route summary exists;
- route label;
- route effect label;
- route-exclusive reward count;
- one readable summary line.

Both the battle completion panel and home summary panel show the line.

## Testing

EditMode tests cover:

- route-exclusive reward appears after a route forms and is categorized as that route;
- selecting it increases the relevant v0.36 combat effect;
- victory summary records the route label, effect, and exclusive reward count.

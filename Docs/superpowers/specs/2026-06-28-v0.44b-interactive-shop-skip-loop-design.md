# v0.44b Interactive Shop Skip Loop Design

## Goal

Fix the v0.44 shop flow so ShopRoom is a room the player can move through, not an automatic purchase popup. The player should interact with a visible shop object to buy, close the shop UI without buying, and continue to the next route even when supplies are insufficient.

## Scope

- Entering ShopRoom no longer opens reward choices automatically.
- ShopRoom still clears immediately because it has no enemies, so the next-route portal remains available.
- A visible `SupplyShopInteractable` object appears in ShopRoom.
- Pressing interact near the shop opens the existing reward-choice UI as a shop purchase draft.
- Shop reward selection still costs supplies. Insufficient supplies keep the shop UI open and show readable feedback.
- Pressing `Esc` or the close button exits the shop draft and returns to route choice.
- Buying one shop reward closes the shop draft, marks the room shop as used, and returns to route choice.

## Architecture

- `RunManager` owns shop state: whether the current room is a shop, whether this shop was used, and whether a shop draft is open.
- `SupplyShopInteractable` is a small world object component. It shows only during ShopRoom and asks `RunManager` to open the shop draft when the player interacts.
- `RoomRewardChoiceUI` gains a shop-only close/skip action.
- Existing `RunSupplyRules`, reward UI, route portal, and companion speech bubble are reused.

## Player Flow

1. Choose and enter a ShopRoom.
2. Move freely in the safe shop room.
3. Either walk to the next-room portal and skip shopping, or approach the shop object.
4. Press `E` near the shop to open purchases.
5. If supplies are too low, close with `Esc` and continue through the route portal.
6. If buying succeeds, the shop closes and the route portal is available again.

## Non-Goals

- No multi-buy shop.
- No shop inventory refresh.
- No persistent currency.
- No complex merchant dialogue.

## Verification

- EditMode tests cover no auto-popup, opening through the shop object, skip/close behavior, insufficient supplies, and one purchase per shop room.
- Unity scene smoke verifies SampleScene and HomeScene play without errors or warnings.

# 版本路线图

## v0.1 玩家横版动作原型

目标：玩家能移动、跳跃、冲刺。

范围：
- 玩家输入读取。
- Rigidbody2D 横向移动。
- 跳跃。
- 冲刺。
- 冲刺期间 `IsDashing`。
- 短暂无敌 `IsInvincible`。

不做：
- 攻击。
- 敌人。
- AI 队友。
- QTE。

完成标准：
- 玩家可在测试场景中左右移动、跳跃、冲刺。
- 移动参数可在 Inspector 调整。

## v0.2 敌人和伤害系统

目标：玩家能攻击木桩敌人，敌人能追击和攻击玩家。

范围：
- `HealthComponent`。
- `DamageInfo`。
- 玩家一次近战攻击检测。
- DummyEnemy 测试对象。
- 基础敌人 Idle、Chase、Attack、Dead 状态。
- 敌人死亡事件。

不做：
- 连招。
- 技能。
- 精英怪。
- Boss。
- AI 队友。

完成标准：
- 玩家可对敌人造成伤害并击杀。
- 敌人可追击玩家并造成伤害。

## v0.3 AI 队友跟随与支援攻击

目标：AI 队友能稳定跟随玩家并攻击附近敌人。

范围：
- 队友半悬浮跟随。
- 距离过远平滑追赶。
- 超过阈值瞬移到玩家附近。
- 队友感知附近敌人。
- 队友攻击最近目标。

不做：
- QTE。
- 对话。
- 复杂寻路。
- 复杂技能。

完成标准：
- 队友能稳定跟随玩家。
- 队友能对敌人造成支援伤害。

## v0.4 AI 不能击杀敌人的保护阈值

目标：AI 能帮忙，但不能替玩家通关。

范围：
- Companion 来源伤害限制。
- 普通敌人最多被 AI 打到 20% 血量。
- 到达阈值时广播补刀请求。
- 玩家伤害不受限制。

不做：
- UI 提示。
- 正式对话。
- 特殊敌人例外规则。

完成标准：
- AI 无法击杀普通敌人。
- 玩家可以正常完成击杀。

## v0.5 QTE 连携系统

目标：AI 根据战机发起 QTE 请求，玩家响应后产生结果。

范围：
- `QTEManager`。
- `CompanionQTERequester`。
- QTE 持续时间。
- Success、WrongInput、Ignored 三种结果。
- Debug 输出结果。

不做：
- 正式 UI。
- LLM。
- 复杂技能演出。

完成标准：
- QTE 可以开始、倒计时、判定三种结果并广播事件。

## v0.6 信赖值、好感度和记忆系统

目标：QTE 和玩家选择能改变 AI 关系。

范围：
- `CompanionRelationship`。
- Trust 和 Affection，范围 0 到 100。
- 隐藏评价标签。
- QTE 结果影响关系值。
- 关系变化事件。

不做：
- 存档。
- LLM。
- 正式家园。

完成标准：
- QTE 结果能改变关系值。
- 关系变化可被 UI 或 Debug 观察到。

## v0.7 基础房间闯关流程

目标：打通最小肉鸽房间流程。

范围：
- `RunManager`。
- `RoomManager`。
- BattleRoom、SafeRoom、ShopRoom、EliteRoom 类型。
- 战斗房生成敌人。
- 敌人全部死亡后触发 RoomCleared。
- 简单 UI 或 Debug 模拟下一房间选择。

不做：
- 复杂地图生成。
- 奖励系统。
- 商店细节。

完成标准：
- 能连续进入房间、清怪、选择下一房间。

## v0.8 濒死保护与分歧事件房

目标：实现 AI 陪伴感最强的核心差异化系统。

范围：
- `BondRescueSystem`。
- 玩家受到致命伤害时触发死亡前检查。
- Trust 达标且本局未触发时拦截死亡。
- 玩家血量设为 1。
- 进入 BranchEventRoom。
- 临时显示三个选项：挽救、直接离开、直面挑战。

不做：
- 正式演出。
- 家园后续。
- 复杂剧情分支。

完成标准：
- 关系达标时 AI 可救援玩家一次。
- 救援后进入分歧事件房流程。

## v0.9 闯关途中动态交流

目标：AI 能根据玩家表现和关卡状态进行本地模板交流。

范围：
- `DialogueTriggerManager`。
- `LocalDialogueTemplate`。
- 监听房间清怪、连续受击、连招失败、QTE 结果。
- 战斗中显示短句。
- 清怪后弹出三个回复选项。
- 回复影响 Trust、Affection 和隐藏评价。
- 对话冷却。

不做：
- LLM。
- 语音。
- 大段剧情。

完成标准：
- AI 能在合适事件后用本地模板发言。
- 玩家回复能影响关系。

## v1.0 家园结算与基础技能树

目标：把单局行为沉淀为局外成长。

范围：
- 家园结算界面。
- 关系总结。
- 本局关键事件摘要。
- 基础技能树。
- 初版局外成长数据。

不做：
- 复杂建造。
- 复杂养成。
- 语音。

完成标准：
- 单局结束后能看到 AI 关系反馈和基础成长选择。

## v1.1 接入 LLM 对话

目标：用 LLM 替换部分本地模板，让 AI 表达更自然。

范围：
- LLM 适配层。
- 本地事件上下文转结构化 Prompt。
- LLM 返回台词、选项、记忆摘要。
- 本地规则校验 LLM 输出。
- 失败时回退本地模板。

不做：
- 让 LLM 直接控制战斗。
- 让 LLM 直接写入存档。
- 让 LLM 判定伤害、死亡、QTE。

完成标准：
- 对话可在本地模板和 LLM 输出之间切换。
- LLM 失败时游戏仍可正常运行。
